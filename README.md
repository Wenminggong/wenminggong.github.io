# Academic Homepage Maintenance Guide

This repository contains the source code for a personal academic homepage built with Jekyll and deployed as a static GitHub Pages site.

Language: English | [中文文档](docs/README-zh.md)

The current site separates content from presentation. Most routine updates only require editing a few focused files:

- Site/profile configuration: `_config.yml`
- Homepage introduction: `_pages/includes/intro.md`
- Publications: `_data/publications.yml`
- Education and experience: `_pages/includes/others.md`
- Homepage composition: `_pages/about.md`
- Publication card and visitor map styles: `assets/css/main.scss`

Do not edit `_site/` directly. It is generated by Jekyll during local builds and deployment.

All examples below use placeholder information such as `Alex Chen`, `Example University`, and `example-user`. Replace them with your own real information when maintaining the site.

## Local Preview

Install Ruby and Bundler first if they are not available on your machine.

From the project root:

```bash
bundle install
bundle exec jekyll serve --livereload --host 127.0.0.1 --port 4000
```

Then open:

```text
http://127.0.0.1:4000/
```

If port `4000` is occupied:

```bash
bundle exec jekyll serve --livereload --host 127.0.0.1 --port 4001
```

Then open:

```text
http://127.0.0.1:4001/
```

Notes:

- Restart the Jekyll server after changing `_config.yml`.
- Markdown, YAML, and SCSS changes usually reload automatically.
- The repository also contains `run_server.sh`, but the explicit `bundle exec jekyll serve --livereload ...` command is easier to debug.

## Homepage Structure

The homepage entry is `_pages/about.md`. It combines the visible homepage sections in this order:

```liquid
{% include_relative includes/intro.md %}
{% include_relative includes/pub.md %}
{% include_relative includes/others.md %}
{% include visitor-map.html %}
```

Normally, you do not need to edit `_pages/about.md` unless you want to reorder, add, or remove whole sections.

## Profile And Site Settings

Edit `_config.yml` for global settings and sidebar profile information.

Example:

```yml
title: "Alex Chen"
description: "Ph.D. Candidate at Example University."
repository: "example-user/example-user.github.io"

author:
  name: "Alex Chen"
  avatar: "images/profile.jpg"
  location: "Example City, Country"
  googlescholar: "https://scholar.google.com/citations?user=EXAMPLE_ID"
  email: "alex.chen@example.edu"
  github: "example-user"
  linkedin: "alex-chen-example"
  orcid: "https://orcid.org/0000-0000-0000-0000"
```

Common updates:

- Change sidebar avatar: place the new image in `images/`, then update `author.avatar`.
- Change contact links: update the corresponding `author` fields.
- Add a personal website: set `author.uri`.
- Change the sidebar description: update `description`.
- Change the browser/site title: update `title`.

After editing `_config.yml`, restart the local Jekyll server.

## About Me

Edit `_pages/includes/intro.md`.

This section is best for:

- Current academic position
- Supervisor and research group
- Research interests
- A short research-topic bullet list

Example:

```md
# About Me
Hello, I am Alex Chen. I am currently a Ph.D. candidate in the Department of Robotics at Example University.

My research interests lie in robotics, learning-based control, and human-robot interaction. I currently focus on:

* Human-Robot Collaboration
* Learning for Dynamics, Control, and Planning
```

Keep this section concise so the homepage stays easy to scan.

## Education And Experience

Edit `_pages/includes/others.md`.

Example:

```md
# Education
- *2024.09 - Present*, Ph.D., Department of Robotics, Example University.
- *2021.09 - 2024.06*, M.S., Department of Automation, Sample Institute of Technology.

# Experience
- *2024.03 - 2024.08*, Research Assistant, Example Robotics Lab.
- *2020.07 - 2021.08*, Software Engineer, Example Tech Inc.
```

Guidelines:

- Use `Present` for ongoing items.
- Keep date formatting consistent, such as `YYYY.MM - YYYY.MM`.
- Avoid trailing whitespace at the end of lines.

## Publications

Publication content is managed in `_data/publications.yml`.

Do not manually maintain the publication list in `_pages/includes/pub.md`. That file only renders the publication include:

```md
# Publications

{% include publications.html %}
```

### Publication Entry Format

Each publication uses this structure:

```yml
- id: example-paper
  visible: true
  category: journal
  title: "An Example Paper Title for Academic Homepage Maintenance"
  url: "https://doi.org/10.0000/example"
  authors: "<strong>Alex Chen</strong>, Taylor Lee, and Jordan Smith"
  venue: "Journal of Example Robotics"
  year: "2026"
  media:
    path: "/papers/figures/example_paper_framework.png"
    alt: "Framework overview for the example paper"
  links:
    - label: "Code"
      url: "https://github.com/example-user/example-paper"
    - label: "PDF"
      url: "/papers/example_paper.pdf"
```

Field meanings:

- `id`: Unique identifier. Use lowercase English words and hyphens.
- `visible`: Use `true` to show the paper and `false` to keep it hidden.
- `category`: Use `journal`, `conference`, or `preprint`.
- `title`: Paper title.
- `url`: Main paper link, such as IEEE, ScienceDirect, arXiv, DOI, or project page.
- `authors`: Author list. Use `<strong>Your Name</strong>` to bold your name.
- `venue`: Journal, conference, workshop, or preprint venue.
- `year`: Publication year.
- `media.path`: Figure or preview image shown in the publication card.
- `media.alt`: Accessible description of the image.
- `links`: Buttons shown below the publication metadata.

### Add A New Publication

1. Put the PDF in `papers/`, for example:

   ```text
   papers/example_paper.pdf
   ```

2. Put the preview figure in `papers/figures/`, for example:

   ```text
   papers/figures/example_paper_framework.png
   ```

3. Add a new entry to `_data/publications.yml`.

4. Preview locally and check that the image, PDF, and external links work.

### Hide Or Draft A Publication

Keep the entry but set:

```yml
visible: false
```

This preserves the data without showing the publication on the homepage.

### Publication Images

Recommended formats:

- Use `.png` for framework diagrams, system diagrams, and screenshots.
- Use `.jpg` for photos or image-heavy figures.
- Avoid using `.pdf` directly as the publication card preview.

If you have a PDF figure such as:

```text
papers/figures/system_framework.pdf
```

export it to:

```text
papers/figures/system_framework.png
```

Then reference the PNG in `_data/publications.yml`:

```yml
media:
  path: "/papers/figures/system_framework.png"
```

## Visitor Map

The visitor map is implemented as an optional Flag Counter embed. It is disabled by default.

Configuration lives in `_config.yml`:

```yml
visitor_map:
  enabled: false
  provider: "flagcounter"
  embed_html: ""
```

To enable it:

1. Generate your own embed HTML from Flag Counter.
2. Paste the exact generated HTML into `embed_html`.
3. Set `enabled: true`.
4. Restart the Jekyll server.

Example:

```yml
visitor_map:
  enabled: true
  provider: "flagcounter"
  embed_html: >
    <a href="https://info.flagcounter.com/example"><img src="https://s01.flagcounter.com/example/bg_FFFFFF/txt_000000/border_CCCCCC/columns_4/maxflags_12/viewers_0/labels_0/pageviews_0/flags_0/percent_0/" alt="Flag Counter"></a>
```

Do not invent a Flag Counter ID manually. Use the code generated by Flag Counter so visits are counted under your own counter.

## Optional Integrations

These options are disabled by default in `_config.yml`.

### MathJax

Use this only if pages need LaTeX math rendering:

```yml
mathjax: true
```

Keep it `false` for a faster homepage if no formulas are shown.

### Google Analytics

Only fill this if you have a Google Analytics measurement ID:

```yml
google_analytics_id: "G-XXXXXXXXXX"
```

If this field is empty, the Google Analytics script is not loaded.

### Google Scholar Citation Stats

The original template supports automated Google Scholar citation stats, but it is disabled here:

```yml
google_scholar_stats_enabled: false
google_scholar_stats_use_cdn: true
```

Only enable it if you also configure the crawler workflow and confirm that the generated `google-scholar-stats` branch exists.

## Navigation

Top navigation is configured in `_data/navigation.yml`.

Current anchor pattern:

```yml
main:
  - title: "About Me"
    url: "/#about-me"
  - title: "Publications"
    url: "/#publications"
  - title: "Education"
    url: "/#education"
  - title: "Experience"
    url: "/#experience"
```

If you rename a section heading, update the corresponding anchor here.

## Styling

The main custom styles are in `assets/css/main.scss`.

Important publication-card classes:

- `.publication-list`
- `.publication-card`
- `.publication-card__media`
- `.publication-card__body`
- `.publication-card__title`
- `.publication-card__links`

Visitor map classes:

- `.visitor-map`
- `.visitor-map__embed`

Prefer small, scoped style changes. Avoid broad theme edits unless necessary.

## Pre-Publish Checklist

Before committing or pushing updates:

```bash
git status
git diff --check
bundle exec jekyll build
```

Also manually check:

- The homepage loads locally.
- Publication images are visible and not distorted.
- PDF/code/video links work.
- Navigation anchors scroll to the right sections.
- Visitor map is hidden unless intentionally enabled.
- `_config.yml` changes are reflected after restarting Jekyll.

## Commit And Deploy

After checking the site:

```bash
git add .
git commit -m "update homepage"
git push
```

GitHub Pages will deploy the site from the repository according to its Pages settings.

## Troubleshooting

### `bundle: command not found`

Install Ruby and Bundler, then rerun:

```bash
bundle install
```

### Changes to `_config.yml` do not appear

Stop and restart the Jekyll server.

### A publication image does not show

Check:

- The file exists.
- The path starts with `/`, for example `/papers/figures/example.png`.
- The file is committed to the repository.
- The filename case matches exactly.

### A new publication does not show

Check:

- `visible: true`
- `category` is one of `journal`, `conference`, or `preprint`
- YAML indentation uses spaces, not tabs
- `_data/publications.yml` is valid YAML

### The visitor map does not show

Check:

- `visitor_map.enabled: true`
- `visitor_map.embed_html` is not empty
- The pasted HTML is valid
- The local Jekyll server was restarted after editing `_config.yml`
